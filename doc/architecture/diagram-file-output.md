# Diagram File Output

The HTML file writer that produces self-contained diagram pages from Mermaid text generated by visualization tool handlers. Replaces the previous SSE-based live diagram viewer (`DiagramBroadcaster`, `/viewer` endpoint, `/viewer/events` SSE stream).

**Feature**: [Diagram File Output](../features/diagram-file-output.md)
**Related**: [MCP Server](mcp-server.md), [Mermaid Renderer](mermaid-renderer.md)

## HtmlWriter Module

Pure function `write_diagram_html(output_path, title, diagrams)` that writes a self-contained HTML file.

- **Input**: output path, title string, list of diagram dicts (`{mermaid: str, label: str | None}`)
- **Output**: writes HTML file, returns the path written
- **Template**: HTML page with mermaid.js from CDN, dark theme, client-side `mermaid.render()` calls
- **Single diagram**: one rendered section
- **Multiple diagrams**: separate sections with label headings (for `visualize_proof_sequence`)
- **Overwrites**: any existing file at output_path

The function is stateless and has no dependencies beyond file I/O. Mermaid text is embedded safely in the HTML (special characters escaped).

## Integration

Each visualization handler calls `write_diagram_html` as a fire-and-forget side effect after generating Mermaid text. The parameter `diagram_dir: Path | None` replaces the previous `broadcaster` parameter:

| Tool | Diagrams passed to writer |
|------|--------------------------|
| `visualize_proof_state` | Single diagram: `[{mermaid: text, label: None}]` |
| `visualize_proof_tree` | Single diagram: `[{mermaid: text, label: None}]` |
| `visualize_dependencies` | Single diagram: `[{mermaid: text, label: None}]` |
| `visualize_proof_sequence` | Multiple diagrams: `[{mermaid: step_text, label: tactic_label}, ...]` |

Title format per tool:

| Tool | Title |
|------|-------|
| `visualize_proof_state` | `Proof State: {session_id} step {step_index}` |
| `visualize_proof_tree` | `Proof Tree: {proof_name}` |
| `visualize_dependencies` | `Dependencies: {name}` |
| `visualize_proof_sequence` | `Proof Sequence: {session_id}` |

When `diagram_dir is None`, no file is written. Exceptions from the writer are caught and logged at WARNING, not propagated to the MCP response.

## Container Configuration

No port mapping changes needed — port 3000 still serves MCP over SSE transport. The HTML file appears in the bind-mounted project directory, directly accessible from the host file system.
